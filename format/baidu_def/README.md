## 概述

百度手机输入法使用的二进制词库格式。

## 文件结构

### 文件头部 (0x00 - 0x6D)

文件头部共 109 字节 (0x6D)，包含编码首字母索引表：

| 偏移量 | 大小  | 类型   | 描述                          |
| ------ | ----- | ------ | ----------------------------- |
| 0x00   | 1字节 | uint8  | 可能是最大码长标识            |
| 0x01   | 4字节 | uint32 | 'a'开头编码的累计字节数       |
| 0x05   | 4字节 | uint32 | 'a'+'b'开头编码的累计字节数   |
| ...    | ...   | ...    | ...                           |
| 0x69   | 4字节 | uint32 | 'a'-'z'开头编码的总累计字节数 |

**索引计算方式：**
- 索引范围：0x60-0x7A (对应ASCII字符 '`' 到 'z')
- 每个4字节值表示从文件开始到该字母的累计数据长度
- 用于快速定位特定首字母的词条区域

### 词条数据区 (0x6D - EOF)

从偏移量 0x6D 开始存储实际的词条数据，每个词条的结构如下：

```
+----------+----------+----------+----------+----------+
| 编码长度 | 词字节长 |   编码   |   词汇   | 结尾标志 |
| 1字节    | 1字节    | N字节    | M字节    | 4字节    |
+----------+----------+----------+----------+----------+
```

#### 词条结构详解

1. **编码长度** (1字节)
   - 存储编码字符串的字节长度
   - 例如："abcd" 的长度为 4

2. **词字节长度** (1字节)
   - 存储词汇的UTF-16LE编码字节数
   - 包含字符串结尾的null终止符

3. **编码** (N字节)
   - ASCII编码的形码字符串
   - 可能包含'='号，'='后的数字为重码数，解析时可以忽略
   - 例如："abcd" 或 "abcd=3"

4. **词汇** (M字节)
   - UTF-16LE编码的中文词汇
   - 包含null终止符

5. **结尾标志** (4字节)
   - 未知的4个零字节 (0x00)
